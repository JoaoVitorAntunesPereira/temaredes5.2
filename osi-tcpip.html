Qual a diferença entre o modelo OSI e o TCP/IP?

A diferença entre o modelo OSI e o TCP/IP que o modelo OSI é de base única, um modelo conceitual que serve para criar outros , e o modelo TCP/IP temos na prática , combinando camadas do OSI em uma algumas poucas versões.

Modelo OSI

O modelo OSI é um modelo de rede de computadores e significa conexão de sistema aberto . É um modelo conceitual com o objetivo de padronizar a comunicação numa rede. Ele é dividido nas seguintes camadas:

  1. Camada física
  2. Camada de rede
  3. Camada de enlace de dados
  4. Camada de sessão
  5. Camada de transporte
  6. Camada de aplicação
  7. Camada de apresentação

Física

Nessa camada temos especificações de transmissão de dados a nível elétrico, de transmissão de bits sem estruturação. É aqui que se fala de cabos de cobre, fibra ótica, adaptadores, etc.

Enlace de dados

Aqui temos uma camada que corrige os erros da camada anterior, podendo controlar e controlar o fluxo de transmissão de dados.

Rede

Essa camada verifica os dados e quem deve quem-los invejosos. Aqui já podemos falar de IP ( internet protocol ), podendo analisar a origem, o destino e definir qual o caminho dos dados.

Transporte

Como o próprio nome já indica, essa camada se refere ao gerenciamento do transporte dos dados, provenientes da camada de rede. Aqui podemos citar o TCP ( protocolo de controle de transmissão ).

Sessão

É nessa camada que estabelecemos comunicação entre computadores, com aplicações diferentes, que passam a ficar sincronizados.

Apresentação

São “traduzidos para um formato em que podem ser usados ​​os dados e transportados para a camada seguinte”.

Aplicação

Finalmente, é na aplicação de dados que são utilizados em aplicações que nós usamos para usar. Aqui você já pode pensar no protocolo HTTP( hypertext transfer protocol ), por exemplo.

Modelo TCP/IP

Trata-se de um conjunto de protocolos de comunicação entre computadores, e o nome de dois protocolos que já falamos aqui. Esses conjuntos de camadas, basicamente, são divididos em 4 camadas (por vezes divididos em 5 camadas):

  1. Camada de física
  2. Camada de rede
  3. Camada de transporte
  4. Camada de aplicação

Física

É nesse nível de comunicação que lidamos com comunicação a nível de hardware, sendo ela responsável por endereçar e trabalhar os funções de gerenciamento de tráfego e velocidade.

Rede

Aqui os dados são anexados ao IP do computador que os invejosos e do destinatário.

Na suíte de protocolos para a internet, o IP executa a tarefa básica de levar pacotes de dados da origem para o destino. O protocolo IP pode transmitir para diferentes protocolos de níveis mais altos, esses protocolos são identificados por um único número de protocolo IP.

Transporte

Essa camada pode resolver problemas de confiabilidade e também determinaram para qual aplicação eles serão dados.

Aplicação

Enfim, na camada de aplicação podemos interagir com os dados que chegam, através de aplicações.

Então é isso, veja um pouco de cada modelo para entender a diferença entre os dois. Se você quiser dissecar o que acontece em cada camada de cada modelo, você pode ler o livro Redes de Computadores do Tanenbaum.
